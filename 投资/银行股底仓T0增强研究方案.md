# 银行股底仓 + T+0 增强 —— 研究方案

> 创建：2026-05-11 | 更新：2026-05-11（明确为实盘辅助，非自动交易）
> 定位：辅助人工决策的信号与分析系统，不直接接入实盘

---

## 写在前面：这个系统到底是干什么的

**规则回顾：** A 股是 T+1 结算。真正的"T+0"是指——你手上**已经持有**某个股票（底仓），当天盘中再买入一笔，等价格拉上去之后，从**底仓里卖出同等数量**。收盘时持股数不变，赚到一笔日内差价。

所以这个系统**不是**帮你自动下单的程序。它的任务更简单也更实用：

> 在交易时段告诉你"XX 银行现在跌到了什么位置，如果你手上有底仓，可以考虑做一次回转了"——然后等盘后给你统计数据，告诉你类似条件下你的历史胜率是多少。

**用户与系统的关系：**
```
实时数据 → 信号计算 → 提示建议 → 你来看 → 你自己决定做不做 → 成交后记录结果
                         ↑                            ↓
                    （系统只算，不做）           （反馈记录供系统学习）
```

## 一、研究问题

### 核心问题（调整后）

1. **选股**：手上有底仓应该配哪几只银行股？多只银行股都持仓时，每个品种各配多少？
2. **触发**：盘中某个银行股跌到什么位置，值得做一次 T+0 回转？
3. **止盈**：买入后涨多少出掉？盘中怎么盯，挂什么价？
4. **止损**：如果买入后继续跌，什么时候认输？
5. **统计**：历史上类似信号出现后的后续走势如何？我的成功率有多高？
6. **风控**：全仓做几次 T+0 比较合理？什么情况下今天最好别动？

### 边界说明

- 系统跑在你自己电脑上（已确认可以跑 Python）
- 数据源：东方财富 API（盘中实时或近实时）
- 输出形式：终端打印/HTML 看板/通知，你自己看，自己下单
- 你不需要手动记录每笔交易——系统帮你记，盘后可以复盘

## 二、需要读的论文与资料

基调变了。不再需要那些"从日线推断日内路径"的统计学论文，改看实打实的 A 股回转交易经验。

### A 类：银行股本身

| 资料 | 为什么读 |
|------|---------|
| **《云蒙——银行股策略回测》**（聚宽社区，2016） | 银行股基本面的经典打分框架（ROE + 核心资本充足率 + 不良/拨备）。用来解决"选哪几只银行股做底仓"的问题。里面有详细的打分公式、调仓规则、回测结果 |
| **雪球/集思录上银行股投资者的讨论**（持续浏览） | 银行股是高股息 + 低波动的品种，做底仓和 T+0 的交易者不少。看看他们用什么指标、怎么判断出手时机。很多经验是论文里没有的 |

### B 类：回转交易实战经验

| 资料 | 为什么读 |
|------|---------|
| **QuantsPlaybook**（GitHub: hugo2046）中的择时策略 | 重点关注 RSRS 择时（判断银行板块处于什么趋势）、低延迟趋势线（快速识别盘中价格触底）。这些指标可以直接用到实时信号计算中 |
| **光大证券 RSRS 择时系列**（2017-2019 金工研报） | 阻力支撑相对强度指标。用在 T+0 上就是——一个股票日内跌了多少才算"超卖"值得买？RSRS 给了一个量化框架 |
| **广发证券 低延迟趋势线与交易择时**（2017） | 低延迟趋势线适合短线判断。在分时图级别上快速识别趋势转向，比均线更敏感 |
| **华泰证券 CSCV 回测过拟合**（2019） | 信号策略最容易踩的坑：把随机噪声当成规律。CSCV 帮你识别"这个信号到底是真的还是运气好" |

### C 类：理论背景（选读，不用全读）

| 资料 | 为什么读 |
|------|---------|
| **Ang et al. (2006)** "The Cross-Section of Volatility and Expected Returns" | 低波动异象为什么存在——解释了银行股长期持有为什么有超额。给你的底仓策略一个理论底气 |
| **Baker, Bradley & Wurgler (2011)** "Understanding the Low-Volatility Anomaly" | 实操层面的解读，读这一篇就够了，不用读 Ang 2006 原文 |

### 阅读建议

```
先快速扫一遍云蒙的银行股文章（半天）
然后浏览 GitHub 上 QuantsPlaybook 里择时策略的 README（半天）
接着看 RSRS 和低延迟趋势线的几篇研报摘要（一天）
最后跑第一个信号计算脚本——同时边跑边看该补什么
```

别在论文上耗太多时间。这个系统是给你实盘用的，跑起来比读透更重要。

## 四、要研究的策略与信号

不需要"分层递进模型"。只需要回答四个问题，每个对应一部分信号逻辑：

---

### 问题 1：底仓选哪些银行股？

> 你不可能 16 只银行股全部持有。选几只，怎么挑？

| 方法 | 逻辑 | 操作 |
|------|------|------|
| 云蒙打分 | 每季报后按 ROE + 核心资本充足率 + 不良率/拨备覆盖率打分，选前 4 名 | 每季度调一次仓 |
| 股息率排名 | 每年按股息率从高到低排，选前 3~5 名 | 每年调一次 |
| 简单等权 | 持有 3~5 只最大的银行股，等权分配 | 几乎不调 |

研究目标：跑一下这三种方式的历史表现，看看哪个更适合做底仓。输出是一个建议的"底仓组合"。

---

### 问题 2：什么时候该做一次 T+0？

> 这是系统的核心模块。盘中需要实时计算的信号。

#### 信号 A：日内跌幅触发
最简单也最常用。当某只银行股当日跌幅超过某个阈值时，提示"可以考虑买入做回转"。

需要研究的参数：
- 买入阈值：跌 1%、跌 1.5%、跌 2%、跌 2.5%？
- 触发基准：相对昨日收盘？相对今日开盘？相对过去 5 日均价？

不同银行股波动率不同，阈值应该跟着走：
- 大行（工行、建行、农行）：日常振幅 ~1.5%，阈值设在 1.0%~1.2% 比较合理
- 股份行（招行、兴业、平安）：日常振幅 ~2.5%，阈值可以设到 1.5%~2.0%
- 城商行（宁波、南京）：日常振幅 ~3%+，阈值设到 2.0%~2.5%

#### 信号 B：RSRS 日内超卖
RSRS 原本是日线择时指标，可以改成分时版本：
- 计算过去 N 个交易日的最高价和最低价序列
- 对每日的（当日最高 / 当日最低）做滚动回归
- 当 R² 显著且斜率下降到某个阈值时，说明价格已经接近支撑位

#### 信号 C：低延迟趋势线穿越
用广发证券的低延迟趋势线方法：
- 计算当前价格的快速下跌速度
- 当价格跌到快速趋势线以下一定幅度时，触发"超卖买入"信号

#### 信号 D：成交量配合
- 如果下跌伴随着缩量（比前几日平均成交量低），说明是正常回调而非恐慌性抛售，做 T+0 更安全
- 如果下跌伴随着放量，说明有机构在出货，最好等一等

---

### 问题 3：买入后什么价格卖出？

| 方法 | 逻辑 |
|------|------|
| 固定止盈 | 买入价 + 0.5% / 0.8% / 1.0% 就出 |
| 动态止盈 | 买入后价格反弹到当日均线之上，或反弹幅度超过下跌幅度的 50% |
| 时间止盈 | 持有时间超过 30 分钟/60 分钟，不管盈亏都出（T+0 不适合持仓过久） |
| 移动止盈 | 买入后价格上行，将止盈线上移，锁定利润 |

卖出是 T+0 里最难的部分。常见亏法：赚了 0.3% 没出，回吐到亏 0.5% 才割。

需要研究：你个人的交易风格更适合哪种止盈方式。系统可以提供多种方案的建议，但最终决策权在你自己手里。

---

### 问题 4：风控怎么做？

| 风控项 | 实现方式 |
|--------|---------|
| 每笔止损 | 买入后跌超 0.5%（可调）就提示止损，不要扛 |
| 单日次数 | 每只银行股每天最多做 1~2 次 T+0，避免频繁交易 |
| 总资金限制 | 每天用于 T+0 的总金额不超过底仓市值的 10%~20% |
| 市场环境 | 银行板块若处于明显的下跌趋势（如 MA60 向下 + 连续 3 日阴线），减少或暂停 T+0 操作 |

## 五、验证与优化方法

### 历史回放（代替传统回测）

不做"全区间数值模拟"，改为**历史行情回放**：

1. 取过去 N 年的银行股日线数据（含分时数据更好）
2. 在每一个交易日，系统模拟"如果一个人坐在电脑前看到这个信号，他会怎么操作"
3. 记录每次模拟操作的盈亏、胜率、平均持有时间
4. 统计出不同阈值组合下的表现

这样既能看到策略的历史表现，又不需要假设"日内价格走什么路径"——因为日线的开盘价/收盘价就是真实价格。

具体做法：
- 用分钟级数据的话，可以模拟到分钟级别的触发
- 用日线数据的话，只能统计"当天触发后收盘价比买入价高/低"——相对粗糙，但也可以作为参考
- 最优方案：用东方财富 API 获取 5 分钟或 15 分钟 K 线数据做回放

### 实盘记录（更有价值）

系统上线后实时记录每一笔 T+0：
```
| 日期 | 股票 | 触发信号 | 买入价 | 卖出价 | 盈亏% | 持有时间 | 备注 |
```

累积到 30~50 笔之后，可以做统计分析了：
- 整体胜率
- 不同信号类型的胜率差异
- 不同银行股的 T+0 收益差异
- 不同时间段的胜率差异（上午 vs 下午、月初 vs 月底）

这些数据比任何回测都更有说服力，因为它们来自**你自己的实际交易**。

## 六、实施路线图

### 阶段一：数据基础 + 信号计算脚本（1 周）

1. 确定银行股底仓名单（先定 3~5 只作为实验对象）
2. 搭建东方财富 API 实时数据获取脚本
3. 实现"日内跌幅触发"信号的基本逻辑
4. 写一个简单脚本：每 5 分钟拉一次数据，计算触发条件，打印建议

```
示例输出：
[10:35] 招商银行 现价 42.30  当日跌幅 -1.86%  ← 触发买入信号（阈值 -1.5%）
        建议：买入底仓量的 10%，目标卖出价 42.65 (+0.83%)
        止损价：42.10 (-0.47%)
```

### 阶段二：增加信号种类 + 盘后记录（1 周）

1. 实现 RSRS 日内超卖信号
2. 实现低延迟趋势线信号
3. 实现成交量辅助判断
4. 搭建 SQLite 交易记录表
5. 写一个手动录入窗口（或从交易所成交数据自动读取）

### 阶段三：历史回放验证（1~2 周）

1. 获取 3~5 年的银行股分钟级 K 线数据
2. 跑回放模拟，统计历史信号表现
3. 优化阈值参数
4. 输出观察报告

### 阶段四：可视化与复盘（1 周）

1. 把每笔 T+0 标在 K 线图上（Lightweight Charts）
2. 生成每周/每月复盘报告（胜率、盈亏分布、热点图）
3. 迭代信号参数

## 七、几个需要想清楚的取舍

**1. 日频 T+0 还是周频 T+0？**
银行股日内振幅有限，做一次 T+0 赚 0.5%~1% 就算不错了。如果你盯盘时间有限，可以放宽到"买入后持有 2~3 天等反弹出"，本质上变成"底仓 + 波段"策略。这和 T+0 不同，但更适合有全职工作的人。

**2. 系统提醒 = 你操作，这个节奏适合你吗？**
如果盘中提醒发出后你来不及看盘，信号就废了。可以考虑把提醒频率降低、条件收紧——只发"高置信度"的信号，宁少勿多。

**3. 记不记自己的交易记录？**
记了才有优化基础。哪怕最初几周只记一个简表，也比不记好十倍。系统可以做这个事，但你得养成"成交后告诉系统"的习惯。
