# 知识图谱模块（元图谱）

## 定位

采用元图谱方案（A方案）：从现有 `[[]]` 双向链接中提炼关系类型并打标签，不建独立图谱目录。在现有文件"关联概念"区域给链接加关系标注。

## 关系类型

```
上游 → 下游
技术 → 产品
产品 → 产业
产业 → 公司
产业 → 政策
技术 → 技术（替代/互补/迭代）
产业 → 产业（交叉/上下游/竞争）
```

## 标注格式

在现有文件的"关联概念"区域，将原本的纯链接改为带关系标注的链接：

```
## 关联概念

- 上游:: [[A股产业研究库/12 术语库/HBM]]
- 下游:: [[A股产业研究库/03 产业链图谱/AI产业链/AI服务器]]
- 替代路线:: [[A股产业研究库/12 术语库/SiC]]
- 互补技术:: [[A股产业研究库/12 术语库/先进封装]]
- 关联产业:: [[A股产业研究库/03 产业链图谱/半导体产业链/总纲]]
```

## 图谱关系总索引

本文件维护所有已标注的关系列表，按产业链分区，作为全库关系图的文本索引。

## 原则

- 所有关系必须具备来源依据
- 禁止主观臆测建立关系
- 标注逐链推进，优先完成AI产业链试点
