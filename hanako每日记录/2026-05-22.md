### OpenHanako Server 部署（腾讯云 150.158.3.58）

从零在腾讯云 Ubuntu 上部署 OpenHanako。遇到的重难点：

**模型加载为 0 的问题：** 根因有两个层次。第一，`scripts/launch.js` 写死了 `HANA_HOME = ~/.hanako-dev`，而 scp 过来的配置在 `~/.hanako`。第二，`auth.json` 被 `_removeApiKeyProviderAuthEntries()` 清空后，Pi SDK 的 `ModelRegistry.getAvailable()` 检测不到 API Key 返回 0 个模型。最终通过创建软链 `.hanako-dev → .hanako` 解决路径问题，并修改 `model-manager.js` 强制刷新 registry。

**其他修复：** `agent-manager.js` 中 `log.error` 不是函数导致真正的错误被吞掉，改为 `log`。
