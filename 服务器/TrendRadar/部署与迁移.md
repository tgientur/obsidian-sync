## 安装过程中的关键问题

**核心瓶颈：代理问题**

- 云服务器无法连接外网 → `git clone` 出错
- Docker 也需要配置代理 → `docker compose pull` / `docker compose build -p` 出错

解决方案见 [[../SSH隧道与代理|SSH 隧道与代理]]。

## 阿里云 Docker 镜像加速

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://8aikki2b.mirror.aliyuncs.com"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker

# 验证
sudo docker info | grep -A 5 "Registry Mirrors"
```

## 迁移至新服务器

```bash
# 原服务器：停止容器并备份
sudo docker compose down
cat docker-compose.yml | grep -A 5 "volumes:"
cd /home/root
tar -czf trendradar_backup.tar.gz TrendRadar

# 传输到新服务器
scp trendradar_backup.tar.gz root@150.158.3.58:/root/

# 新服务器：解压并启动
tar -xzf /root/trendradar_backup.tar.gz
cd /TrendRadar/docker
sudo docker compose up -d

# 验证
sudo docker compose ps
sudo docker logs trendradar --tail 30
```
