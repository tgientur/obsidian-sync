## 📋 完整步骤

### 前提条件
- 本地电脑已安装 OpenSSH 客户端
- 本地电脑 Clash 已开启，且 **Allow LAN** 已打开
- 阿里云服务器的 SSH 端口（22 或你修改的端口）可正常连接

---

### 第一步：确认本地 Clash 端口

打开 Clash，查看 **HTTP 代理端口**（通常是 7890）：
- 一般 Clash 默认：HTTP 代理 `127.0.0.1:7890`
- 确认 **Allow LAN** 已开启

---

### 第二步：在本地电脑 PowerShell 中建立反向隧道

```powershell
# 基础命令（替换为你的服务器 IP 和端口）
ssh -NfR 7890:localhost:7890 -p 22 root@你的服务器IP

# 推荐：加心跳包防止断连
ssh -o ServerAliveInterval=60 -NfR 7890:localhost:7890 -p 22 root@你的服务器IP
```

**执行后**：
- 如果没有错误提示，直接返回新的一行 → ✅ 成功
- 如果提示输入密码，输入密码后回车（屏幕不显示是正常的）

**参数说明**：
| 参数 | 作用 |
|------|------|
| `-N` | 不执行远程命令 |
| `-f` | 后台运行 |
| `-R 7890:localhost:7890` | 反向隧道：服务器 7890 → 本地 7890 |
| `-p 22` | 服务器的 SSH 端口 |
| `-o ServerAliveInterval=60` | 每 60 秒发心跳包，防止断连 |

**⚠️ 注意**：保持这个 PowerShell 窗口**不要关闭**，可以最小化。

---

### 第三步：在服务器上测试隧道是否正常

**新开一个 PowerShell 窗口**（或通过其他终端），SSH 登录到服务器：

```bash
ssh root@你的服务器IP -p 22
```

登录后执行测试命令：

```bash
# 测试 1：查看端口是否在监听
netstat -tlnp | grep 7890
# 应该看到 127.0.0.1:7890 在 LISTEN 状态

# 测试 2：通过代理访问 Google（验证代理是否真正工作）
curl -I -x http://127.0.0.1:7890 https://www.google.com
# 期望输出：HTTP/1.1 200 Connection established
```

如果 `curl` 测试返回 `HTTP/1.1 200`，说明代理完全正常。

---

### 第四步：设置服务器环境变量（让 RSS 抓取走代理）

在服务器上执行：

```bash
# 临时设置（当前 SSH 会话有效）
export http_proxy=http://127.0.0.1:7890
export https_proxy=http://127.0.0.1:7890

# 测试是否生效
echo $http_proxy
# 应显示 http://127.0.0.1:7890
```

**永久生效**（可选，避免每次登录都要设置）：

```bash
echo 'export http_proxy=http://127.0.0.1:7890' >> ~/.bashrc
echo 'export https_proxy=http://127.0.0.1:7890' >> ~/.bashrc
source ~/.bashrc
```

---

### 第五步：测试 RSS 抓取是否成功

```bash
# 手动测试 RSS 抓取（雅虎财经）
curl -I https://finance.yahoo.com/news/rssindex

# 如果返回 200，说明 RSS 可以正常访问了
```

---

### 第六步：重启 TrendRadar 容器

```bash
cd /home/admin/TrendRadar/docker
sudo docker compose restart
```

## ❌ 常见问题排查

| 问题 | 可能原因 | 解决方法 |
|------|---------|---------|
| `Connection refused` | 本地 Clash 没开或端口不对 | 检查 Clash 是否运行，端口是否为 7890 |
| `Connection timed out` | 服务器 SSH 端口不通或防火墙 | 检查阿里云安全组是否开放 22 端口 |
| `Permission denied` | SSH 密码错误或密钥问题 | 确认密码，或用密钥登录 |
| 代理测试超时 | 本地 Clash 节点问题 | 切换 Clash 节点重试 |
| RSS 测试超时 | 代理未生效 | 确认 `export http_proxy` 已设置 |
